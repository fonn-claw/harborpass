- The Check In button in the top bar needs to open the wizard in dispatch-board.tsx. Since layout.tsx is a server component and dispatch-board.tsx is a client component, the button must be wired through the client tree.
- Approach: Pass a "Check In" button as the `actions` prop to TopBar. This button is rendered in the top bar. But since the wizard state lives in dispatch-board.tsx, use a different approach:
  - Create a small client component `CheckInTrigger` that renders a teal button with the check-in icon (/assets/icon-check-in.svg). It uses a custom event or a shared context to communicate with dispatch-board.
  - Simpler approach: Use a global custom event. CheckInTrigger dispatches `window.dispatchEvent(new CustomEvent("open-checkin-wizard"))`. DispatchBoard listens for this event via useEffect and opens the wizard with no pre-selected stay or slip.
- In layout.tsx: import CheckInTrigger (client component), pass it as `actions={<CheckInTrigger />}` to TopBar.
- CheckInTrigger component: "use client", renders `<Button className="bg-teal hover:bg-teal/90 text-white gap-2"><img src="/assets/icon-check-in.svg" ... /> Check In</Button>`
