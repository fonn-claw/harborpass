**Create src/app/board/actions.ts ("use server"):**
- `checkInGuest(data: CheckInFormData)` server action:
  1. Validate input with checkInSchema.parse(data)
  2. If `existingGuestId` AND `existingStayId` (pre-booked flow):
     - Generate credentials: `generateGateCode()`, `generateWifiPassword(data.guestName.split(" ").pop()!)`
     - Update existing stay: set status "checked_in", slipId, gateCode, wifiPassword, showerTokens: 3, showerTokensUsed: 0
     - Update slip status to "occupied"
  3. If new walk-up (no existingGuestId):
     - Insert new guest record (name, vesselName, loa, beam, draft, phone, email)
     - Generate credentials
     - Look up nightly rate from pricing table: query pricing where category="slip" and name matches slip size
     - Calculate expectedDeparture: checkIn + expectedNights days
     - Insert stay: guestId, slipId, checkIn=now, expectedDeparture, status "checked_in", gateCode, wifiPassword, showerTokens: 3, nightlyRate
     - Update slip status to "occupied"
  4. Call `revalidatePath("/board")`
  5. Return `{ success: true, gateCode, wifiPassword, showerTokens: 3 }`
  6. Wrap everything in try/catch â€” on error return `{ success: false, error: message }`

CRITICAL: Do NOT use `db.transaction()` â€” Neon HTTP driver doesn't support it. Use sequential inserts. On error, the demo data can be re-seeded.

CRITICAL: Return credentials BEFORE revalidatePath takes effect. The wizard captures the return value in the client, so credentials are available even as the board re-renders.
